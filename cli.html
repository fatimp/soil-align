Утилита командной строки

Утилита командной строки имеет следующий интерфейс (выводится при запуске без аргументов): 

Abrupt ending of the input: argument #<ARGUMENT {1100444043}> missing
Usage: soil-align [-v|--verbose] [-t|--threads N] [-w|--workspace-side S]
                  [-O|--transform-output m.npy] [-o|--output out.npy]
                  [--rotation-constraint AXIS] [-s|--scaling]
                  [-b|--background-color COLOR] [--dist-ratio C]
                  [--fit-error E] [--ransac-iterations M]
                  reference source

Description of flags and options
--ransac-iterations M           Number of RANSAC iterations
--fit-error E                   The maximal allowed fit error to treat a sample as inlier
--dist-ratio C                  Controls what we consider a match
-b|--background-color COLOR     Background color
-s|--scaling                    Add uniform scaling to the model
--rotation-constraint AXIS      Constrain rotation to be around only one axis (0, 1 or 2)
-o|--output out.npy             Output file name for a transformed image (.npy or .raw)
-O|--transform-output m.npy     Output file name for a transform matrix (.npy)
-w|--workspace-side S           Side of a workspace which is cut from center of the input images
-t|--threads N                  Number of threads to use
-v|--verbose                    Be verbose

Два обязательных аргумента reference и source — это два входных изображения в формате numpy массивов (.npy файлы) или raw. Soil-align попробует сопоставить изображение source с reference. Для работы также необходимы опции -m или -o(или обе сразу). Первая выводит матрицу преобразования, переводящую source в reference, а вторая — трансформированное изображение source таким образом, чтобы его элементы максимально совпадали с reference. Тип элемента (dtype в numpy) входных массивов должен быть uint8. Опция -w позволяет вырезать область из центра входных массивов и искать ключевые точки в ней, применяя потом найденное преобразование на весь исходный массив, таким образом экономя время и память. Параметры -s и -r управляют тем, какая модель будет использована для поиска преобразования. Флаг -s включает поиск коэффициента масштабирования, а -r позволяет ограничить вращение одной из осей. Остальные опции относятся к настройкам алгоритмов 3D SIFT и RANSAC. Рекомендуется обратиться к литературе для понимания их значения.

Для работы с raw форматом (не рекомендуется!) нужно создать файл метаданных с расширением s-exp на каждый входной raw файл, имеющий вид: 

(имя-файла-в-кавычках размерность-0 размерность-1 размерность-2)

например 

("foo.raw" 1000 1000 600)

Ожидается, что raw файл будет хранить данные в row-major порядке, один беззнаковый октет на воксель. Для записи преобразованного изображения в raw формат нужно указать выходной файл с расширением raw. Размерность выходного массива соответствует размерности source, поэтому для него файл метаданных не нужен.

Soil-align использует базу данных для сохранения дескрипторов. По умолнчанию база сохраняется в директории ~/.local/share/soil-align/, но путь до базы может быть переназначен переменной окружения SOIL_ALIGN_DB. В данный момент база занимает довольно значительный объём, и рекомендуется вручную удалять её после окончания работы с датасетом.
